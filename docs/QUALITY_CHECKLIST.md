# Quality Checklist

Pre-launch verification checklist for each IGNITE lead magnet. Use this before launching any magnet to ensure quality standards.

---

## How to Use This Checklist

1. **Print or copy** for each magnet
2. **Complete all sections** before launch
3. **Have second team member verify** critical items
4. **Document issues** and resolution
5. **Archive completed checklists** for reference

## Section 1: Brand Alignment

### Colors
- [ ] Fire Orange (#FF4D00) used for primary CTAs
- [ ] Charcoal (#2C3539) used for body text/dark backgrounds
- [ ] White (#FFFFFF) used appropriately for contrast
- [ ] No unapproved colors present
- [ ] Color contrast meets accessibility standards (WCAG AA)

### Typography
- [ ] Inter font specified throughout
- [ ] Font weights correct (400, 600, 700 only)
- [ ] Typography hierarchy followed (H1: 48px, H2: 36px, etc.)
- [ ] Text readable at all sizes
- [ ] Line height appropriate (1.6 for body text)

### Voice & Tone
- [ ] Direct, no-fluff language throughout
- [ ] Authoritative, conviction in statements
- [ ] Action-oriented (focus on what to DO)
- [ ] No corporate jargon
- [ ] No hedging language ("maybe", "possibly", "try")
- [ ] No passive voice
- [ ] Toby Potter style evident
- [ ] Tone appropriate for context (landing page vs. email vs. ad)

**Voice Check Examples:**
- [ ] First 3 sentences get straight to point
- [ ] No paragraph longer than 4 sentences
- [ ] Every section leads to action
- [ ] Excuses called out directly
- [ ] Empowering while challenging

## Section 2: Landing Page Quality

### Technical Functionality
- [ ] HTML validates (use validator.w3.org)
- [ ] Page loads in < 3 seconds
- [ ] Mobile responsive (test on iPhone and Android)
- [ ] All links work correctly
- [ ] Form submits successfully
- [ ] Thank-you page displays after opt-in
- [ ] Facebook Pixel fires correctly
- [ ] No console errors in browser

### Design & Layout
- [ ] Hero section compelling (headline + CTA visible without scrolling)
- [ ] Visual hierarchy clear
- [ ] White space used effectively
- [ ] Images high quality (not pixelated)
- [ ] CTA buttons stand out (Fire Orange)
- [ ] Mobile layout doesn't break
- [ ] Typography readable on mobile

### Copy Quality
- [ ] Headline addresses specific pain point
- [ ] Subheadline clarifies value proposition
- [ ] Benefits clearly stated (not features)
- [ ] Social proof included (if available)
- [ ] Urgency or scarcity element present (appropriate, not manipulative)
- [ ] CTA text action-oriented ("Download Now" not "Submit")
- [ ] No typos or grammatical errors

### Conversion Optimization
- [ ] Form fields minimized (name + email only)
- [ ] CTA above the fold
- [ ] Multiple CTAs throughout page (for scrollers)
- [ ] Friction-reducing elements ("Free", "Instant Access", "No Credit Card")
- [ ] Privacy assurance present
- [ ] Exit-intent popup (optional but recommended)

---

## Section 3: Email Sequence Quality

### Technical Setup
- [ ] All emails loaded into ESP
- [ ] Automation triggers configured correctly
- [ ] Email delays set appropriately (1-2 days between emails)
- [ ] Segmentation tags applied correctly
- [ ] IGNITE pitch email included (Email 5-7)
- [ ] Unsubscribe link present in all emails
- [ ] Physical address included (CAN-SPAM compliance)
- [ ] From name consistent ("Toby Potter" or "IGNITE")
- [ ] Reply-to email active and monitored

### Email Content Quality (Per Email)
- [ ] Subject line compelling (40-60 characters)
- [ ] Preview text optimized (don't repeat subject line)
- [ ] Opening line hooks reader
- [ ] Email length 150-300 words (concise)
- [ ] One primary CTA per email
- [ ] Links tracked for analytics
- [ ] Signature consistent across sequence
- [ ] Mobile-friendly formatting

### Sequence Structure
- [ ] Email 1: Deliver lead magnet + set expectations
- [ ] Email 2: Quick win or insight
- [ ] Email 3: Deeper value + story
- [ ] Email 4: Address objection
- [ ] Email 5: Introduce IGNITE program
- [ ] Email 6: Social proof + testimonials
- [ ] Email 7: Clear CTA to join IGNITE

### Deliverability
- [ ] Domain authentication (SPF, DKIM, DMARC)
- [ ] Spam score < 5 (use Mail Tester)
- [ ] No spam trigger words overused
- [ ] Text-to-image ratio appropriate
- [ ] No suspicious links
- [ ] Test email delivered to inbox (not spam)

---

## Section 4: Facebook Ads Quality

### Ad Account Setup
- [ ] Campaign objective: Lead Generation
- [ ] Conversion event: Lead (not just link click)
- [ ] Pixel installed and firing
- [ ] Budget appropriate ($30-50/day to start)
- [ ] Bidding strategy: Lowest cost
- [ ] Campaign budget optimization enabled

### Audience Targeting
- [ ] Age range: 25-55
- [ ] Geographic targeting appropriate
- [ ] Interests relevant to audience segment
- [ ] Behaviors aligned with target (online shoppers, etc.)
- [ ] Exclusions set (existing customers)
- [ ] Lookalike audience (if available)

### Ad Creative (Per Variation)
- [ ] Image high quality (min 1080x1080)
- [ ] Text overlay < 20% of image area
- [ ] Visual grabs attention (pattern interrupt)
- [ ] Brand colors evident
- [ ] Faces/people shown (if appropriate)
- [ ] Mobile-optimized (square or vertical)

### Ad Copy
- [ ] Headline 40 characters or less
- [ ] Primary text 125 characters or less (mobile-optimized)
- [ ] Opens with hook (question, bold statement, pain point)
- [ ] Benefit-driven (not feature-driven)
- [ ] CTA clear ("Download", "Take Quiz", "Get Access")
- [ ] No policy violations (check Facebook ad policies)
- [ ] No exaggerated claims or guarantees
- [ ] Appropriate for audience (no "you" language in B2B cold traffic)

### Compliance
- [ ] No before/after health claims
- [ ] No income guarantees
- [ ] No targeting based on sensitive attributes
- [ ] Disclaimer if required
- [ ] Brand name included somewhere in ad

## Section 5: Video Script Quality (If Applicable)

### Script Structure
- [ ] Opening hook (first 3 seconds)
- [ ] Problem agitation (seconds 4-15)
- [ ] Solution introduction (seconds 16-30)
- [ ] Value delivery (seconds 31-90)
- [ ] CTA (last 10-15 seconds)
- [ ] Total length 60-120 seconds

### Content Quality
- [ ] Conversational language (not scripted-sounding)
- [ ] Direct to camera instructions clear
- [ ] B-roll suggestions included
- [ ] Text overlay cues provided
- [ ] Music/tone guidance included
- [ ] CTA repeated at end

### Production Readiness
- [ ] Teleprompter-ready formatting
- [ ] Pronunciation guide for difficult words
- [ ] Shot list provided
- [ ] Graphic insert points marked
- [ ] Caption requirements specified

## Section 6: PDF/Tool Quality

### PDF Design (If Workbook/Ledger/Framework)
- [ ] Brand colors throughout
- [ ] Inter font used consistently
- [ ] Cover page professional
- [ ] Table of contents (if >10 pages)
- [ ] Page numbers
- [ ] Headers/footers with branding
- [ ] High resolution (min 300 DPI for print)
- [ ] File size optimized (<10MB)

### PDF Content
- [ ] Instructions clear and actionable
- [ ] Fillable fields (if interactive)
- [ ] Examples provided where helpful
- [ ] White space for writing (if workbook)
- [ ] No orphan headings (heading at bottom of page)
- [ ] Consistent formatting throughout
- [ ] CTA to IGNITE program included (last page)

### Tool Functionality (If Quiz/Calculator/Generator)
- [ ] Logic tested thoroughly
- [ ] All paths lead to result
- [ ] Results personalized and valuable
- [ ] Mobile-friendly interface
- [ ] Load time acceptable
- [ ] Error handling present
- [ ] Thank-you/result page clear

### Email Platform Integration
- [ ] Landing page connected to ESP
- [ ] Opt-in triggers automation
- [ ] Lead tagged correctly
- [ ] Welcome email fires immediately
- [ ] Sequence continues as scheduled
- [ ] Test opt-in completed successfully

### Analytics Tracking
- [ ] Google Analytics installed (if used)
- [ ] Facebook Pixel installed and verified
- [ ] Conversion events tracked
- [ ] UTM parameters in ad links
- [ ] Goal/event tracking configured
- [ ] Dashboard created for monitoring

- [ ] Lead score assigned (if using)
- [ ] Sales team notified for high-intent leads

## Section 8: Legal & Compliance

### Privacy & Data Protection
- [ ] Privacy policy link present
- [ ] GDPR compliance (if serving EU)
- [ ] Clear explanation of data use
- [ ] Option to opt-out provided
- [ ] Data storage compliant with regulations

### Marketing Compliance
- [ ] CAN-SPAM compliant (physical address, unsubscribe)
- [ ] No false or misleading claims
- [ ] Testimonials compliant (results not typical disclosed)
- [ ] Affiliate disclosures (if applicable)
- [ ] Terms of service link present

### Intellectual Property
- [ ] No copyrighted images without license
- [ ] Music licensed (if video)
- [ ] Competitor names used appropriately (no trademark infringement)
- [ ] Citations for statistics/research

---

## Section 9: User Experience Testing

### Real User Test
- [ ] Complete entire funnel yourself
- [ ] Have non-team member complete funnel
- [ ] Test on mobile device
- [ ] Test on tablet
- [ ] Test in different browsers (Chrome, Safari, Firefox)
- [ ] Time from opt-in to lead magnet delivery
- [ ] Email sequence delivers correctly
- [ ] All links work from emails

### Feedback Collection
- [ ] Test user found value in lead magnet
- [ ] Test user understood next steps
- [ ] Test user encountered no technical issues
- [ ] Test user felt brand was professional
- [ ] Test user clear on IGNITE offer

## Section 10: Performance Benchmarks

### Expected Performance (Week 1)

**Landing Page:**
- [ ] Conversion rate target set (25-40%)
- [ ] Baseline traffic goal set
- [ ] A/B test plan prepared (if <25%)

**Email Sequence:**
- [ ] Open rate target set (40-60%)
- [ ] Click rate target set (10-20%)
- [ ] Unsubscribe rate threshold set (<2%)

**Facebook Ads:**
- [ ] CTR target set (1.5-3%)
- [ ] Cost per lead target set ($3-8)
- [ ] Daily budget set
- [ ] Scaling plan prepared (if performance exceeds targets)

**IGNITE Conversions:**
- [ ] Lead-to-customer conversion target set (2-5%)
- [ ] Customer acquisition cost threshold set (<$150)
- [ ] Sales follow-up process defined

## Section 11: Launch Readiness

### Team Alignment
- [ ] All team members aware of launch date/time
- [ ] Roles and responsibilities clear
- [ ] Monitoring schedule defined
- [ ] Escalation process for issues defined
- [ ] Communication protocol established (Slack, etc.)

### Backup Plans
- [ ] Backup landing page version saved
- [ ] Backup email sequence exported
- [ ] Alternative email platform identified (if primary fails)
- [ ] Alternative ad account prepared (if primary suspended)
- [ ] Customer support plan for questions

### Post-Launch Monitoring
- [ ] First 2 hours: Active monitoring
- [ ] First 24 hours: Check every 4 hours
- [ ] First week: Daily check-in
- [ ] Metrics dashboard bookmarked
- [ ] Alert thresholds set (if possible)
