# Belief System Audit - Quality Assurance Checklist

## PRE-LAUNCH CHECKLIST

Complete this checklist before launching the Belief System Audit to ensure all components are production-ready and optimized for conversion.

---

## SECTION 1: LANDING PAGE

### Brand Compliance
- [ ] Fire Orange (#FF4D00) used for all CTAs
- [ ] Charcoal (#2C3539) used for body text and dark backgrounds
- [ ] White (#FFFFFF) used appropriately for contrast
- [ ] Inter font loaded correctly (weights: 400, 600, 700)
- [ ] Typography hierarchy followed (H1: 48px/700, H2: 36px/600, Body: 16px/400)
- [ ] Toby Potter voice maintained (direct, authoritative, action-oriented)

### Content Quality
- [ ] Headline clearly states the problem: "Your Beliefs Don't Match Your Behavior"
- [ ] Subheadline includes clear value proposition
- [ ] Problem section resonates with self-aware professionals
- [ ] Solution section explains the belief-behavior gap
- [ ] Statistics/research cited are accurate (70%, 50%, Age 6)
- [ ] All 6 belief pattern types described clearly
- [ ] No typos, grammatical errors, or formatting issues

### Technical Functionality
- [ ] Page loads in <3 seconds (test on GTmetrix/PageSpeed Insights)
- [ ] Mobile responsive (test on iPhone, Android, tablet)
- [ ] All CTA buttons functional and link to quiz
- [ ] Smooth scroll for anchor links works
- [ ] Images optimized (compressed, correct format)
- [ ] Font loading doesn't cause layout shift
- [ ] No console errors in browser inspector

### Conversion Optimization
- [ ] Primary CTA above the fold (visible without scrolling)
- [ ] Multiple CTAs throughout page (hero, after problem, after patterns, final)
- [ ] CTA copy is action-oriented ("START YOUR FREE AUDIT NOW")
- [ ] Social proof element included (if available: "Join 10,000+ professionals")
- [ ] Trust-building elements present ("No tricks. No upsells. Just truth.")
- [ ] Clear time expectation ("8 minutes")
- [ ] Low friction emphasized ("Free," "No credit card")

### SEO & Metadata
- [ ] Page title optimized: "Free Belief System Audit | Discover What's Holding You Back"
- [ ] Meta description compelling and under 160 characters
- [ ] Open Graph tags for social sharing (og:title, og:description, og:image)
- [ ] Favicon present
- [ ] Canonical URL set
- [ ] Schema markup for webpage (optional but recommended)

---

## SECTION 2: ASSESSMENT/QUIZ

### Question Quality
- [ ] All 8 questions use behavior-based language (not hypothetical)
- [ ] Questions reference specific timeframes ("last 30 days," "last 3 months")
- [ ] Questions span multiple domains (career, money, health, relationships)
- [ ] One third-person perspective question included (reduces bias)
- [ ] One emotional trigger question included
- [ ] Answer options are mutually exclusive and comprehensive
- [ ] No leading or judgmental language in questions or answers

### Scoring System
- [ ] Each answer maps to specific belief pattern(s)
- [ ] Pattern frequency calculation formula is correct
- [ ] Primary pattern determined by highest frequency (3+ occurrences)
- [ ] Belief Alignment Score calculation accurate (healthy responses / 8 × 100)
- [ ] Mixed pattern handling included (no score above 2)
- [ ] Secondary influence identified correctly

### User Experience
- [ ] Progress bar shows completion percentage
- [ ] Questions display one at a time (not all at once - overwhelming)
- [ ] "Save and return" option available (captures email after Q3-4)
- [ ] No time limit (creates anxiety)
- [ ] Back button allows answer changes
- [ ] Mobile-friendly interface (large tap targets, readable text)
- [ ] Clear visual feedback when answer selected

### Technical Implementation
- [ ] Quiz platform selected and set up (TypeForm, Interact, Gravity Forms, or custom)
- [ ] Conditional logic working (if applicable)
- [ ] Data validation (all questions answered before submission)
- [ ] Response data captured in database/spreadsheet
- [ ] No data loss on browser refresh (auto-save)
- [ ] GDPR/CCPA compliant (privacy policy linked, consent checkbox)

### Email Capture
- [ ] Email gate positioned optimally (before full results or after teaser)
- [ ] Copy emphasizes value: "Enter your email to receive your complete report"
- [ ] Privacy assurance visible ("We'll never share your email")
- [ ] Email validation (correct format required)
- [ ] Double opt-in confirmation email sent (best practice for deliverability)
- [ ] Welcome email triggers immediately upon submission

## SECTION 4: EMAIL SEQUENCE

### Sequence Setup
- [ ] All 7 emails written and loaded in ESP (ConvertKit, ActiveCampaign, etc.)
- [ ] Correct timing intervals set (Immediate, Day 2, 4, 6, 8, 10, 12)
- [ ] Pattern-specific segments created (6 segments for each belief type)
- [ ] Email 2-4 alternate versions for each pattern loaded
- [ ] Tags applied correctly upon quiz completion (pattern type, alignment score)
- [ ] Automation trigger tested (quiz completion → sequence start)

### Email Content Quality
- [ ] All 7 emails match Toby Potter voice
- [ ] Subject lines compelling and under 50 characters
- [ ] Preview text optimized (doesn't repeat subject)
- [ ] Each email has single, clear CTA
- [ ] Personalization tokens work ([FIRST_NAME], [PATTERN_TYPE], etc.)
- [ ] Pattern-specific emails reference their exact pattern
- [ ] No typos or grammatical errors
- [ ] Links functional and tracked

### Technical Setup
- [ ] Sender domain authenticated (SPF, DKIM, DMARC)
- [ ] "From" name set correctly (Toby Potter / IGNITE)
- [ ] Reply-to address monitored (replies encouraged)
- [ ] Unsubscribe link present and functional (legal requirement)
- [ ] Plain-text version created (not just HTML)
- [ ] Mobile preview checked (50%+ open on mobile)
- [ ] Test sends reviewed for rendering issues

### Compliance & Deliverability
- [ ] CAN-SPAM compliant (physical address in footer)
- [ ] GDPR compliant (clear consent, easy opt-out)
- [ ] No spam trigger words in subject/body
- [ ] Text-to-image ratio appropriate (mostly text)
- [ ] Links not excessive (2-3 max per email)
- [ ] Engagement signals optimized (replies encouraged)

### Daily Checks
- [ ] Monitor quiz completion rate (target: 60-75%)

- [ ] Review Facebook ad performance (CTR, CPL)
- [ ] Verify automation working (random spot checks)
- [ ] Read user feedback/replies
- [ ] Check for technical errors (broken links, 404s)

### Week 1 Analysis
- [ ] Traffic sources analyzed (which ads/channels performing best)
- [ ] Conversion funnel reviewed (where are drop-offs?)
- [ ] Email engagement measured (open rates, click rates)
- [ ] Quiz data analyzed (which questions cause hesitation/abandonment?)
- [ ] Pattern distribution reviewed (are results distributed evenly?)
- [ ] ROI calculated (cost vs. email subscribers vs. IGNITE enrollments)

### Optimization Priorities
Based on Week 1 data, identify:
1. **Biggest bottleneck:** Where are we losing the most people?
2. **Quick wins:** What can we improve in <1 hour that moves the needle?
3. **A/B test:** What should we test first to improve conversion?
